# Usage Guide

## Purpose

Use this guide to produce a complete EMOVEL Launch Stack for one product idea.

The stack is designed to be used in one focused production pass. Each step feeds the next. Do not start with page copy or visual direction before the audience, offer, and pricing are clear.

## Required Input

Before running the stack, collect:

- Product name or working title
- One-sentence product idea
- Target buyer
- Problem solved
- Desired outcome
- Current product format
- Existing proof or credibility
- Competitors or alternatives
- Launch goal
- Delivery constraints
- Preferred price range, if any

## Production Order

1. Audience
2. Offer
3. Pricing
4. Copy
5. Page
6. Visual
7. Funnel
8. Launch plan

## Step 1: Build The Audience Snapshot

Use: `emovel.audience_builder`

Create:

- ICP name
- Avatar summary
- Demographics
- Psychographics
- Daily life snapshot
- Top problems
- Top desires
- Buying objections
- Trusted sources
- Buying behavior
- Emotional triggers
- Language patterns
- Anti-avatar

Quality check:

- The profile should reject most of the market.
- The buyer's language should sound specific.
- The anti-avatar should make the offer sharper.

## Step 2: Build The Offer Stack

Use: `emovel.offer_architect`

Create:

- Core transformation
- Core offer
- Bonus stack
- Value assignments
- Total value summary
- Guarantee
- Urgency mechanism
- Offer name
- One-sentence summary
- Price anchoring narrative

Quality check:

- The offer should sell an outcome, not a bundle of files.
- Bonuses should make the core outcome easier or faster.
- Urgency must be real.
- The guarantee must reduce perceived risk without promising revenue.

## Step 3: Build The Pricing Map

Use: `emovel.pricing_engine`

Create:

- Recommended price
- Tier structure
- Anchor price
- Payment plan
- Bundle option
- Price increase roadmap

Quality check:

- Use no more than three tiers.
- Each tier needs a clear reason to upgrade.
- Payment plans should total 20% to 30% more than the one-time price.
- The price should signal quality.

## Step 4: Draft The Copy System

Use: `emovel.copy_framework`

Create:

- Main headline
- Headline variants
- Problem section
- Solution section
- Benefit copy
- CTA variants
- Objection-handling copy
- Email subject lines or hooks, if needed

Quality check:

- Lead with the reader's problem or desire.
- Keep one clear job per copy block.
- Use direct language.
- Avoid vague claims.

## Step 5: Build The Landing Page Structure

Use: `emovel.page_builder`

Create:

- SEO metadata
- Hero
- Problem section
- Solution section
- Deliverables section
- Benefits section
- Proof section
- Offer stack
- Guarantee
- FAQ
- Closing CTA

Quality check:

- Primary CTA must appear above the fold.
- Page must follow Attention, Interest, Desire, Action.
- FAQ should answer real buying objections.
- Risk reversal must be visible before final CTA.

## Step 6: Build The Visual Brief

Use: `emovel.visual_brief`

Create:

- Brand personality
- Color palette
- Typography
- Visual style descriptor
- Image direction
- Iconography
- Layout principles
- Moodboard directions
- Platform specs
- Do and do not list

Quality check:

- The visual system should match buyer trust signals.
- Keep font choices limited.
- Define what not to do.
- Make the brief useful for a designer or page builder.

## Step 7: Build The Funnel Map

Use: `emovel.funnel_builder`

Create:

- Entry point
- Page sequence
- Email sequence
- Traffic strategy
- Conversion benchmarks
- Upsell and downsell
- Tech stack
- A/B testing priorities

Quality check:

- The funnel must have a defined entry and end state.
- Lead magnets must deliver value before a pitch.
- Offers over $500 should consider a call or application step.
- Retargeting should be planned if paid traffic is used.

## Step 8: Package The Final Output

Final client or product folder should include:

- `README.md`
- `offer.md`
- `product-structure.md`
- `landing-page-copy.md`
- `visual-brief.md`
- `launch-plan.md`
- `usage-guide.md`

For client delivery, combine the strongest sections into one clean PDF or Google Doc.

## Delivery Checklist

- [ ] Audience is specific and useful.
- [ ] Offer is outcome-led.
- [ ] Price has clear rationale.
- [ ] Page has one primary CTA.
- [ ] Copy handles objections.
- [ ] Visual brief gives concrete direction.
- [ ] Funnel has measurable conversion goals.
- [ ] Launch plan includes content and outreach.
- [ ] Guarantee is clear and realistic.
- [ ] Next action is obvious.

## Operator Notes

Do not overcomplicate v1. The product wins by creating usable clarity fast.

The stack should feel like a premium launch strategist compressed into a repeatable system. Every output should help the buyer do one of three things:

1. Explain the offer.
2. Build the page.
3. Start selling.
