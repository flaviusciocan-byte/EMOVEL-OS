# ux-audit

## Objective

The product experience must make EMOVEL Prompt Systems feel like a practical operating layer for AI work, not a generic prompt pack.

## Primary UX Jobs

- Explain the difference between prompts and prompt systems.
- Show the included systems clearly.
- Make the $197 price feel like a high-leverage workflow upgrade.
- Emphasize repeatability, quality control, and production use.
- Avoid abstract AI hype.

## Recommended Page Flow

1. Hero with direct repeatability promise.
2. Problem: inconsistent AI output comes from inconsistent prompting systems.
3. Framework explanation: input, context, output, QA.
4. Included systems grid.
5. Use cases by buyer type.
6. Pricing and what is included.
7. FAQ.
8. Final CTA.

## Conversion Risks

- Product may be confused with a prompt list unless the system architecture is shown.
- Buyers may wonder if it works with their AI tool. Clarify tool-agnostic usage.
- The offer needs concrete examples of output format and quality checks.

## UX Requirements

- Use diagrams or compact tables to show prompt system anatomy.
- Keep examples scannable.
- Include a clear "not just prompts" section.
- CTA should be simple: Get Prompt Systems.
