# ux-audit

## Objective

The product experience must make the Landing Template Pack feel like a practical deploy-ready asset, not just a design inspiration bundle.

## Primary UX Jobs

- Show that templates are actual Next.js/Tailwind implementation assets.
- Explain who each template is for.
- Make deployment and customization feel simple.
- Make $97 feel like a small cost compared with rebuilding pages manually.
- Emphasize mobile-first and conversion structure.

## Recommended Page Flow

1. Hero with deploy-ready promise.
2. Problem: blank-page rebuilds waste launch time.
3. Template previews and use cases.
4. Included components and sections.
5. Tech stack and deployment flow.
6. Pricing.
7. FAQ.
8. Final CTA.

## Conversion Risks

- Buyers may assume it is only static screenshots. Clarify code files.
- Buyers may worry it is hard to customize. Show the content/component structure.
- Buyers may ask whether shadcn/ui is required. State what is included and optional.

## UX Requirements

- Use visual previews or file structure panels.
- Show template names and best use cases.
- Include a clear deployment path.
- CTA label: Get The Template Pack.
